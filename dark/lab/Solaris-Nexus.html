Solaris Nexus

An exploration of interactive cosmic systems

Type Web Project
Status In Progress
Tech HTML • CSS • JS
Duration Jan – Feb

The Story

I've always been fascinated by the quiet vastness of space. I wanted a project not just built with standard components, but one that felt like a living breathing system.

The spark for Solaris Nexus was a simple curiosity: "How expressive can vanilla CSS and JavaScript get before it starts begging for an heavy external library?" I wanted to prove that immersion doesn't require a 2MB framework. I wanted to tell a story through layout and micro-interactions.

The Build

Concept

A central cosmic theme serving as a playground for performance-focused vanilla JS physics.

Design Decisions

Opting for soft glassmorphisms and orbit-based timelines instead of rigid squares. Using a deep space gradient instead of pure black to retain depth.

Challenges

Keeping the requestAnimationFrame loop performant when rendering continuous star trails and ensuring mobile layout didn't break during heavy animations.

What I Learned

Sometimes the browser itself is your best tool. Pure mathematics combined with CSS transforms can out-perform heavy canvas libraries if constrained properly.

Reflection

What worked: The glassmorphism and subtle glowing node elements turned out beautifully. The aesthetic immediately communicates the "cosmos" vibe without being cliché.

What didn't: Initial logic for star arrays was too heavy; calculating thousands of floats on the CPU caused hiccups on low-end devices.

What I'd improve: I want to migrate the most intensive background starfield rendering directly into WebGL for absolute fluid 120fps motion.
